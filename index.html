基于STM32的智能浇花系统

Smart Plant Watering System Based on STM32

STM32F103C8T6 ESP8266 OneNet 微信小程序 OLED

项目概述

本项目设计并实现了一套基于STM32F103C8T6微控制器的智能浇花系统，具备土壤湿度自动检测、智能浇水决策、水位监测报警及远程数据交互功能。

系统架构

系统采用模块化设计，以STM32F103C8T6为核心控制单元，各功能模块划分如下：

模块分类组成功能
输入模块土壤湿度传感器、水位传感器、4路独立按键采集土壤湿度、检测水箱水位、接收用户操作
控制核心STM32F103C8T6ADC+DMA采集、滞回控制算法、MQTT协议栈、Flash存储
输出模块OLED显示屏、继电器+水泵、LED报警灯+蜂鸣器数据显示、自动浇灌、水位报警
通信模块ESP8266 (WiFi) → OneNet云平台 → 微信小程序远程数据交互、阈值下发

系统供电链：12V DC → TPS54302DDA (5V) → AMS1117-3.3 (3.3V)

STM32 下位机代码详解

STM32代码采用Keil MDK开发，基于STM32标准外设库（V3.5.0），前后台架构（裸机运行）。整体代码分为以下目录：

目录说明
Main/主函数 main.c、中断服务、外设配置头文件
BSP/板级支持包：ADC、按键、串口、延时、蜂鸣器/LED/水泵驱动
APP/应用层：OLED驱动、字库（8×16 ASCII点阵）
Onenet/ESP8266 AT指令驱动、MQTT协议栈 (MqttKit)、cJSON解析、OneNet业务层
Libraries/STM32标准外设库（CMSIS + FWlib）
Project/Keil uVision5 项目文件 (.uvprojx)

主循环调度 (main.c)

代码采用前后台架构，主循环中轮询调度6个任务：

  • OneNet数据接收处理 — 检查ESP8266缓冲区是否有下行数据，若有则通过OneNet_RevPro()解析，支持远程修改湿度阈值
  • 按键扫描 — Handle_Keys()，支持4个独立按键
  • 水位检测与报警 — 实时读取PA2引脚的数字电平，水位过低时点亮LED并上报报警状态
  • 电机（水泵）控制 — Motor_Control_Check()，滞回控制：湿度低于阈值+400次循环计时（约4秒）后关泵
  • 定时采集与上报 — 每100次循环（约1秒）读取一次ADC值并发布到OneNet
  • 心跳保活 — 每3000次循环（约30秒）发送一次MQTT PINGREQ保持连接

土壤湿度采集 (bsp_adc.c/h)

通过STM32的ADC1通道1（PA1）采集土壤湿度传感器的模拟电压值，采用DMA循环传输模式，无需CPU干预实时刷新ADC值。

// ADC配置：独立模式、连续转换、DMA循环传输 static void ADCx_Mode_Config(void) { // DMA: 外设地址→内存(ADC_ConvertedValue[1]) // ADC: 独立模式、连续转换、SCAN+CONT使能 // 通道1 (PA1), 采样周期55.5个周期 }

ADC原始值通过公式转换为湿度百分比：

// ADC值 → 湿度百分比 float ADC_To_Humidity(uint16_t adc_val) { float range = (ADC_WET_VALUE - ADC_DRY_VALUE); float percent = (adc_val - ADC_DRY_VALUE) / range * 100.0f; return percent; } // ADC_DRY_VALUE = 0 (干燥时最小值) // ADC_WET_VALUE = 2160 (浸水时最大值)

OLED显示 (bsp_oled.c/h + bsp_oledfont.h)

0.96寸OLED显示屏，分辨率128×64，通过软件模拟I2C协议与STM32通信（SCL: PB6，SDA: PB7）。

采用双缓冲机制：先在内存 OLED_GRAM[8][128] 中操作，最后通过OLED_Refresh()一次性刷新到屏幕。

// 显示逻辑：根据是否处于"设置模式"切换显示 static void OLED_Display(float humidity, uint8_t threshold) { OLED_Clear(); if (setting_mode) OLED_Display_Setting(threshold); // 编辑模式：显示 EDIT THRESHOLD [ XX ]% else OLED_Display_Normal(humidity, threshold); // 正常模式：显示 Humidity / Threshold OLED_Refresh(); }

按键处理 (bsp_key.c/h)

4个独立按键（KEY1:PB0，KEY2:PA7，KEY3:PA3，KEY4:PA5），采用状态机+10ms延时消抖：

  • KEY1: 进入/退出阈值设置模式
  • KEY2: 阈值 +5%（设置模式下）
  • KEY3: 阈值 -5%（设置模式下）
  • KEY4: 加载Flash中保存的阈值

WiFi与云平台通信

ESP8266驱动 (esp8266.c/h)

通过AT指令控制ESP8266模块，实现串口WiFi透明传输。初始化流程：

  1. AT → 检测模块
  2. AT+CWMODE=1 → Station模式
  3. AT+CWDHCP=1,1 → 使能DHCP
  4. AT+CWJAP="SSID","PASSWORD" → 连接WiFi路由器
  5. AT+CIPSTART="TCP","mqtts.heclouds.com",1883 → 连接OneNet MQTT服务器

MQTT协议栈 (MqttKit.c/h)

项目使用轻量级MQTT协议栈（Li Zhi Qu开发，V1.6），支持CONNECT、SUBSCRIBE、PUBLISH、PING等功能。

关键通信参数：

参数值
MQTT Brokermqtts.heclouds.com:1883
设备发布主题$sys/{product_id}/{device_name}/thing/property/post
设备订阅主题$sys/{product_id}/{device_name}/thing/property/set
回复主题$sys/{product_id}/{device_name}/thing/property/set_reply
心跳间隔约30秒（PINGREQ）

OneNet业务层 (onenet.c/h)

实现设备登录、属性上报、下行命令解析等功能。上报数据格式为JSON：

// 上报数据示例 {"id":"123", "params":{ "Humidity":{"value":68.5}, "Threshold":{"value":50}, "alarm":{"value":false} }} // 下行解析：通过cJSON从发布消息中提取"Threshold"字段并更新

阈值掉电存储

系统将湿度阈值存储在STM32内部Flash的0x0800FC00地址（最后一页），使用魔数0xA5A5校验有效性，确保重启后阈值不丢失。

static void Threshold_Save(uint8_t val) { FLASH_Unlock(); FLASH_ErasePage(FLASH_SAVE_ADDR); FLASH_ProgramHalfWord(FLASH_SAVE_ADDR, FLASH_MAGIC); // 写魔数 0xA5A5 FLASH_ProgramHalfWord(FLASH_SAVE_ADDR + 2, val); // 写阈值 FLASH_Lock(); }

微信小程序作为远程监控端，通过OneNet云平台的HTTP API获取设备数据并向设备下发指令。代码使用原生微信小程序框架开发。

数据交互流程

系统采用微信小程序 → OneNet HTTP API → OneNet云平台 → MQTT → STM32设备的数据链路：

  1. 微信小程序每3秒通过OneNet HTTP API轮询设备属性（GET /thingmodel/query-device-property）
  2. STM32设备每1秒通过MQTT发布土壤湿度、阈值、报警状态到OneNet
  3. 用户在小程序中修改阈值时，通过POST /thingmodel/set-device-property 下发指令
  4. OneNet通过MQTT下行通道将新阈值推送到STM32设备
  5. STM32解析下行数据后更新本地阈值并存入Flash

硬件设计与PCB

硬件原理图与PCB使用嘉立创EDA设计（.epro2文件）。系统硬件包含：

  • 主控：STM32F103C8T6最小系统板（72MHz, 64KB Flash, 20KB SRAM）
  • 电源：12V DC输入 → TPS54302DDA (12V→5V) → AMS1117-3.3 (5V→3.3V)
  • 水泵驱动：NPN三极管SS8050 + 续流二极管，控制12V直流水泵
  • 传感器：土壤湿度传感器（TLC555，电阻式模拟输出）+ 水位传感器（DICS-D，数字量）
  • 显示：0.96寸OLED 128×64，I2C接口
  • 通信：ESP8266串口WiFi模块

PCB文件为嘉立创EDA标准格式 (嘉立创文件.epro2)，可直接在嘉立创EDA中打开查看或打样。

项目总结

本设计完成了一套完整的智能浇花嵌入式系统：

  1. 硬件上完成了原理图设计与PCB布局打样，集成了多种传感器与执行器；
  2. 软件上实现了STM32嵌入式程序（ADC采集、OLED显示、按键交互、MQTT通信、Flash存储）；
  3. 云平台方面打通了OneNet MQTT数据通道，实现设备属性上报与远程控制；
  4. 用户端开发了微信小程序，支持湿度实时查看、水位报警、植物预设阈值一键切换等功能。
