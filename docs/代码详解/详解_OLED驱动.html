OLED 显示驱动模块详解 (SSD1306)

文件：APP/bsp_oled.c APP/bsp_oled.h  |  基于 STM32 智能浇花系统

模块概述 硬件连接 软件I2C时序 命令/数据写入 双缓冲机制 字符显示 初始化序列 刷新流程 完整代码

1. 模块概述

本模块驱动一颗 0.96寸 OLED 显示屏，分辨率为 128×64 像素，控制器为 SSD1306。通信接口采用 软件模拟 I2C（位冲模式），占用两个 GPIO 引脚：

引脚分配： PB6 → SCL（时钟线），PB7 → SDA（数据线）

模块核心设计亮点：

  • 双缓冲机制 — 在内存中维护 OLED_GRAM[8][128] 显存缓冲，写入操作先修改缓冲，再通过 OLED_Refresh() 批量刷新到物理屏幕，避免逐点写入造成的闪烁。
  • 软件 I2C — 不依赖 STM32 硬件 I2C 外设，任意两个 GPIO 均可模拟，移植性极强。
  • 8×16 像素字库 — 内嵌 ASCII 字符点阵（从空格到大写 Z），覆盖常用可打印字符。

模块对外提供的主要 API：

  • OLED_Init() — 初始化 GPIO 和 OLED 控制器
  • OLED_ShowChar(x, y, chr) — 在指定位置显示一个字符
  • OLED_Clear() — 清屏（填充 0x00）
  • OLED_Refresh() — 将显存缓冲刷新到屏幕

2. 硬件连接

OLED 模块通过 4 根线连接至 STM32 主控：

STM32 引脚OLED 模块说明
PB6SCLI2C 时钟线
PB7SDAI2C 数据线
3.3VVCC电源正极
GNDGND电源地
注意： SSD1306 的 I2C 地址默认为 0x78（7 位地址 0x3C 左移一位），定义在 bsp_oled.h 第 6 行。如果模块背面 SA0/SDO 引脚拉高，地址可能变为 0x7A。

头文件中通过宏定义将 GPIO 操作集中管理，方便移植到其他引脚：

// SCL: PB6, SDA: PB7
#define OLED_SCL_GPIO_PORT  GPIOB
#define OLED_SCL_RCC        RCC_APB2Periph_GPIOB
#define OLED_SCL_PIN        GPIO_Pin_6

#define OLED_SDA_GPIO_PORT  GPIOB
#define OLED_SDA_RCC        RCC_APB2Periph_GPIOB
#define OLED_SDA_PIN        GPIO_Pin_7

#define OLED_SCL_H  GPIO_SetBits(OLED_SCL_GPIO_PORT, OLED_SCL_PIN)
#define OLED_SDA_H  GPIO_SetBits(OLED_SDA_GPIO_PORT, OLED_SDA_PIN)
#define OLED_SCL_L  GPIO_ResetBits(OLED_SCL_GPIO_PORT, OLED_SCL_PIN)
#define OLED_SDA_L  GPIO_ResetBits(OLED_SDA_GPIO_PORT, OLED_SDA_PIN)

#define OLED_IIC_SDA_READ()  GPIO_ReadInputDataBit(OLED_SDA_GPIO_PORT, OLED_SDA_PIN)

3. 软件 I2C 时序

本模块完全通过 GPIO 软件模拟 I2C 协议，不依赖 STM32 硬件 I2C 外设。时序核心是四个基础函数：

3.1 起始信号 — IIC_Start

当 SCL 为高电平时，SDA 从高切换到低，表示总线起始条件：

void IIC_Start(void)
{
    /* SCL 高电平时, SDA 下降沿 = 启动信号 */
    OLED_SDA_H;
    OLED_SCL_H;
    IIC_Delay();
    OLED_SDA_L;
    IIC_Delay();
    OLED_SCL_L;   // 钳住总线, 开始传输
    IIC_Delay();
}

3.2 停止信号 — IIC_Stop

当 SCL 为高电平时，SDA 从低切换到高，表示总线停止条件：

void IIC_Stop(void)
{
    /* SCL 高电平时, SDA 上升沿 = 停止信号 */
    OLED_SDA_L;
    OLED_SCL_H;
    IIC_Delay();
    OLED_SDA_H;
}

3.3 等待应答 — IIC_WaitAck

主机释放 SDA，拉高 SCL，检测从机是否将 SDA 拉低（ACK）：

uint8_t IIC_WaitAck(void)
{
    uint8_t re;
    OLED_SDA_H;     // CPU 释放 SDA 总线
    IIC_Delay();
    OLED_SCL_H;     // 拉高 SCL, 从机输出 ACK
    IIC_Delay();
    if (OLED_IIC_SDA_READ()) re = 1;  // NACK
    else re = 0;                          // ACK
    OLED_SCL_L;
    IIC_Delay();
    return re;
}

3.4 字节写入 — Write_IIC_Byte

MSB 优先，逐位输出到 SDA 线，每个位在 SCL 上升沿被从机采样：

void Write_IIC_Byte(uint8_t _ucByte)
{
    uint8_t i;
    for (i = 0; i < 8; i++)
    {
        if (_ucByte & 0x80) OLED_SDA_H;
        else                 OLED_SDA_L;
        IIC_Delay();
        OLED_SCL_H;           // 从机在上升沿采样
        IIC_Delay();
        OLED_SCL_L;
        if (i == 7) OLED_SDA_H; // 释放总线, 等待 ACK
        _ucByte <<= 1;
        IIC_Delay();
    }
}
时序要点： 软件 I2C 的速率由 IIC_Delay() 中的空循环次数控制（当前为 10 次），实测约 100-200 kHz，完全满足 SSD1306 的 I2C 最高速率 400 kHz 要求。

4. 命令 / 数据写入

SSD1306 通过控制字节区分后续字节是命令还是显示数据：

操作控制字节后续字节函数
写命令0x00命令码Write_IIC_Command()
写数据0x40像素数据Write_IIC_Data()

Write_IIC_Command

void Write_IIC_Command(u8 IIC_Command)
{
    IIC_Start();
    Write_IIC_Byte(OLED_ADDRESS);    // 0x78 (器件地址 + 写)
    IIC_WaitAck();
    Write_IIC_Byte(0x00);            // 控制字节: 后续为命令
    IIC_WaitAck();
    Write_IIC_Byte(IIC_Command);      // 命令码
    IIC_WaitAck();
    IIC_Stop();
}

Write_IIC_Data

void Write_IIC_Data(u8 IIC_Data)
{
    IIC_Start();
    Write_IIC_Byte(OLED_ADDRESS);    // 0x78
    IIC_WaitAck();
    Write_IIC_Byte(0x40);            // 控制字节: 后续为显示数据
    IIC_WaitAck();
    Write_IIC_Byte(IIC_Data);         // 像素数据
    IIC_WaitAck();
    IIC_Stop();
}
关键区别： 命令写入使用控制字节 0x00，数据写入使用 0x40。SSD1306 根据 D/C# 位（控制字节的 bit-6）决定将接收字节存入命令寄存器还是 GDDRAM。

5. 双缓冲机制

模块在 STM32 内部维护一个 1024 字节 的显存缓冲区：

u8 OLED_GRAM[8][128];  // 8 页 × 128 列
维度取值范围含义
页 (page)0 ~ 7每页对应 8 个像素行（共 64 行）
列 (column)0 ~ 127每列对应一个像素列
每个字节0x00 ~ 0xFFbit-0 对应页内最上一行, bit-7 对应最下一行

工作流程：

  1. 所有绘图操作（OLED_ShowChar、OLED_Fill 等）只修改 OLED_GRAM[][]
  2. 操作完成后调用 OLED_Refresh() 一次性将整个缓冲区刷到屏幕
  3. 刷新通过 I2C 逐页发送 128 字节数据
优势： 避免频繁 I2C 通信造成的画面闪烁和 CPU 占用；写缓冲在 RAM 中极快，刷新时才产生总线流量。

6. 字符显示 — OLED_ShowChar

OLED_ShowChar 是实现文本输出的核心函数。它将一个 ASCII 字符的 8×16 点阵写入显存：

void OLED_ShowChar(u8 x, u8 y, u8 chr)
{
    u8 c, i;
    c = chr - ' ';                    // 计算字库索引 (空格 = 0)
    if(x > 127) return;
    if(y > 6)   return;              // y 是页号, 最多到第 6 页 (留 1 页给下半部分)
    for(i = 0; i < 8; i++)
    {
        if(x + i < 128)
        {
            OLED_GRAM[y][x + i]     = zf[c * 16 + i];       // 上半部分 (第 y 页)
            OLED_GRAM[y + 1][x + i] = zf[c * 16 + i + 8];   // 下半部分 (第 y+1 页)
        }
    }
}
字库布局： 每个字符占 16 字节。前 8 字节对应上半部分 8 行（写入第 y 页），后 8 字节对应下半部分 8 行（写入第 y+1 页）。一个字库项通过 zf[c * 16 + i] 索引。

注意 y 参数的单位是"页"而非像素行，取值范围 0~6。一个字符垂直占用 2 页（16 像素行），总可用高度为 8 页（64 行）。

7. 初始化序列 — OLED_InitController

SSD1306 上电后需要发送一系列配置命令才能正常工作。OLED_InitController() 按标准初始化序列发送命令：

static void OLED_InitController(void)
{
    Write_IIC_Command(0x00);  Write_IIC_Command(0x10);  // 列起始地址低/高字节
    Write_IIC_Command(0x40);                                  // 显示起始行
    Write_IIC_Command(0xB0);                                  // 页起始地址
    Write_IIC_Command(0x81);  Write_IIC_Command(0xFF);  // 对比度: 最大
    Write_IIC_Command(0xA1);                                  // 段重映射 (左右翻转)
    Write_IIC_Command(0xA6);                                  // 正常显示 (非反色)
    Write_IIC_Command(0xA8);  Write_IIC_Command(0x3F);  // 多路复用比: 64 行
    Write_IIC_Command(0xC8);                                  // COM 扫描方向: 从下到上
    Write_IIC_Command(0xD3);  Write_IIC_Command(0x00);  // 显示偏移: 0
    Write_IIC_Command(0xD5);  Write_IIC_Command(0x80);  // 振荡频率
    Write_IIC_Command(0xD8);  Write_IIC_Command(0x05);  // 电荷泵预分频
    Write_IIC_Command(0xD9);  Write_IIC_Command(0xF1);  // 预充电周期
    Write_IIC_Command(0xDA);  Write_IIC_Command(0x12);  // COM 引脚配置
    Write_IIC_Command(0xDB);  Write_IIC_Command(0x30);  // VCOMH 电压
    Write_IIC_Command(0x8D);  Write_IIC_Command(0x14);  // 使能电荷泵
}
命令参数说明
0x00 / 0x10—设置列起始地址为 0
0x40—显示起始行 = 0
0xB0—页起始地址 = 0
0x810xFF对比度设置（最大亮度）
0xA1—段重映射（左右镜像关闭）
0xA6—正常显示模式（非反白）
0xA80x3F多路复用比 = 64（128×64 模式）
0xC8—COM 扫描方向：从 COM[N-1] 到 COM0
0xD30x00显示偏移 = 0
0xD50x80显示时钟分频 / 振荡频率
0xD80x05电荷泵预分频 / 复合周期
0xD90xF1预充电周期
0xDA0x12COM 引脚硬件配置（交替模式）
0xDB0x30VCOMH 电压选择
0x8D0x14使能内部电荷泵（必须！否则无显示）
重要： 命令 0x8D + 0x14 用于使能 SSD1306 内部 DC-DC 电荷泵。如果漏掉此命令，OLED 屏幕将不亮。

8. 刷新流程 — OLED_Refresh

刷新过程分为三步：

  1. 调用 OLED_InitController() 重新发送初始化命令（确保控制器处于已知状态）
  2. 逐页调用 OLED_FlushPage(page) 发送 128 字节数据
  3. 发送 0xAF 命令打开显示

OLED_Refresh

void OLED_Refresh(void)
{
    u8 page;
    OLED_InitController();                    // 重新初始化
    for(page = 0; page < 8; page++)
        OLED_FlushPage(page);                   // 逐页刷新
    Write_IIC_Command(0xAF);                  // 打开显示
}

OLED_FlushPage — 页缓冲 I2C 突发写入

static void OLED_FlushPage(u8 page)
{
    u8 col;
    OLED_Set_Pos(0, page);                     // 定位到目标页
    IIC_Start();
    Write_IIC_Byte(OLED_ADDRESS);
    IIC_WaitAck();
    Write_IIC_Byte(0x40);                    // 数据模式
    IIC_WaitAck();
    for(col = 0; col < 128; col++)
    {
        Write_IIC_Byte(OLED_GRAM[page][col]);  // 连续发送 128 字节
        IIC_WaitAck();
    }
    IIC_Stop();
}

OLED_Set_Pos — 设置页和列地址

void OLED_Set_Pos(u8 x, u8 y)
{
    Write_IIC_Command(0xb0 + y);              // 页地址 (0~7)
    Write_IIC_Command(((x & 0xf0) >> 4) | 0x10);  // 列地址高 4 位
    Write_IIC_Command((x & 0x0f) | 0x00);     // 列地址低 4 位
}
性能： 每页通过一次 I2C 起始/停止条件发送 128+ 字节，8 页共 1024+ 字节。在 ~100 kHz 下刷新时间约 100 ms，人眼不会感知到刷屏过程。

bsp_oled.c

/* bsp_oled.c — OLED 驱动实现 */
#include "bsp_oled.h"
#include "bsp_oledfont.h"

// GPIO 初始化: PB6 SCL, PB7 SDA, 推挽输出 50 MHz
void IIC_GPIO_Config(void)
{
    GPIO_InitTypeDef GPIO_InitStructure;
    RCC_APB2PeriphClockCmd(OLED_SCL_RCC | OLED_SDA_RCC, ENABLE);

    GPIO_InitStructure.GPIO_Pin   = OLED_SCL_PIN;
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;
    GPIO_InitStructure.GPIO_Mode  = GPIO_Mode_Out_PP;
    GPIO_Init(OLED_SCL_GPIO_PORT, &GPIO_InitStructure);

    GPIO_InitStructure.GPIO_Pin   = OLED_SDA_PIN;
    GPIO_Init(OLED_SDA_GPIO_PORT, &GPIO_InitStructure);

    IIC_Stop();  // 总线复位
}

static void IIC_Delay(void)
{
    uint8_t i;
    for (i = 0; i < 10; i++);
}

// I2C 起始: SCL 高时 SDA 下降沿
void IIC_Start(void)
{
    OLED_SDA_H; OLED_SCL_H; IIC_Delay();
    OLED_SDA_L; IIC_Delay();
    OLED_SCL_L; IIC_Delay();
}

// I2C 停止: SCL 高时 SDA 上升沿
static void IIC_Stop(void)
{
    OLED_SDA_L; OLED_SCL_H; IIC_Delay();
    OLED_SDA_H;
}

// 等待从机 ACK: 返回 0=应答, 1=无应答
uint8_t IIC_WaitAck(void)
{
    uint8_t re;
    OLED_SDA_H; IIC_Delay();
    OLED_SCL_H; IIC_Delay();
    if (OLED_IIC_SDA_READ()) re = 1;
    else                       re = 0;
    OLED_SCL_L; IIC_Delay();
    return re;
}

// I2C 写入一个字节 (MSB first)
void Write_IIC_Byte(uint8_t _ucByte)
{
    uint8_t i;
    for (i = 0; i < 8; i++)
    {
        if (_ucByte & 0x80) OLED_SDA_H;
        else                 OLED_SDA_L;
        IIC_Delay(); OLED_SCL_H; IIC_Delay();
        OLED_SCL_L;
        if (i == 7) OLED_SDA_H;
        _ucByte <<= 1; IIC_Delay();
    }
}

// 写命令: 控制字节 = 0x00
void Write_IIC_Command(u8 IIC_Command)
{
    IIC_Start();
    Write_IIC_Byte(OLED_ADDRESS); IIC_WaitAck();
    Write_IIC_Byte(0x00);         IIC_WaitAck();
    Write_IIC_Byte(IIC_Command);  IIC_WaitAck();
    IIC_Stop();
}

// 写数据: 控制字节 = 0x40
void Write_IIC_Data(u8 IIC_Data)
{
    IIC_Start();
    Write_IIC_Byte(OLED_ADDRESS); IIC_WaitAck();
    Write_IIC_Byte(0x40);         IIC_WaitAck();
    Write_IIC_Byte(IIC_Data);     IIC_WaitAck();
    IIC_Stop();
}

u8 OLED_GRAM[8][128];  // 显存缓冲: 8 页 x 128 列

// 设置光标位置 (页/列)
void OLED_Set_Pos(u8 x, u8 y)
{
    Write_IIC_Command(0xb0 + y);
    Write_IIC_Command(((x & 0xf0) >> 4) | 0x10);
    Write_IIC_Command((x & 0x0f) | 0x00);
}

// 全屏填充
void OLED_Fill(u8 fill_Data)
{
    u8 m, n;
    for(m = 0; m < 8; m++)
        for(n = 0; n < 128; n++)
            OLED_GRAM[m][n] = fill_Data;
}

void OLED_Clear(void) { OLED_Fill(0x00); }

// 显示一个 8x16 字符
void OLED_ShowChar(u8 x, u8 y, u8 chr)
{
    u8 c, i;
    c = chr - ' ';
    if(x > 127) return;
    if(y > 6)   return;
    for(i = 0; i < 8; i++)
    {
        if(x + i < 128)
        {
            OLED_GRAM[y][x + i]     = zf[c * 16 + i];
            OLED_GRAM[y + 1][x + i] = zf[c * 16 + i + 8];
        }
    }
}

// SSD1306 初始化命令序列
static void OLED_InitController(void)
{
    Write_IIC_Command(0x00); Write_IIC_Command(0x10);
    Write_IIC_Command(0x40); Write_IIC_Command(0xB0);
    Write_IIC_Command(0x81); Write_IIC_Command(0xFF);
    Write_IIC_Command(0xA1); Write_IIC_Command(0xA6);
    Write_IIC_Command(0xA8); Write_IIC_Command(0x3F);
    Write_IIC_Command(0xC8); Write_IIC_Command(0xD3);
    Write_IIC_Command(0x00); Write_IIC_Command(0xD5);
    Write_IIC_Command(0x80); Write_IIC_Command(0xD8);
    Write_IIC_Command(0x05); Write_IIC_Command(0xD9);
    Write_IIC_Command(0xF1); Write_IIC_Command(0xDA);
    Write_IIC_Command(0x12); Write_IIC_Command(0xDB);
    Write_IIC_Command(0x30); Write_IIC_Command(0x8D);
    Write_IIC_Command(0x14);
}

// 刷新单页到 SSD1306
static void OLED_FlushPage(u8 page)
{
    u8 col;
    OLED_Set_Pos(0, page);
    IIC_Start();
    Write_IIC_Byte(OLED_ADDRESS); IIC_WaitAck();
    Write_IIC_Byte(0x40);         IIC_WaitAck();
    for(col = 0; col < 128; col++)
    {
        Write_IIC_Byte(OLED_GRAM[page][col]);
        IIC_WaitAck();
    }
    IIC_Stop();
}

// 刷新整屏: 初始化 → 逐页刷 → 开显示
void OLED_Refresh(void)
{
    u8 page;
    OLED_InitController();
    for(page = 0; page < 8; page++)
        OLED_FlushPage(page);
    Write_IIC_Command(0xAF);  // Display ON
}

// OLED 初始化入口: GPIO → 清屏 → 刷新
void OLED_Init(void)
{
    IIC_GPIO_Config();
    OLED_Clear();
    OLED_Refresh();
}
