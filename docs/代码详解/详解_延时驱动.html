1. 模块概述

本模块基于 SysTick（系统滴答定时器）实现精确的微秒级和毫秒级延时，采用轮询 COUNTFLAG 的方式完成延时等待。

时钟配置： SysTick 时钟源选择 HCLK/8，即 72 MHz ÷ 8 = 9 MHz，每个计数周期 = 1/9 μs ≈ 0.111 μs。

2. 延时初始化 — Delay_Init

配置 SysTick 定时器的时钟源并计算延时系数。

void Delay_Init(void)
{
    // 选择时钟源为 HCLK/8 (CTRL 寄存器的 bit2 清零)
    SysTick->CTRL &= ~(1 << 2);

    // 1 us 需要的计数次数：9 MHz / 1,000,000 = 9
    UsCount = 9;

    // 1 ms 需要的计数次数：9 * 1000 = 9000
    MsCount = UsCount * 1000;
}
系数推导： SysTick 时钟 9 MHz，每秒计数 9,000,000 次。1 μs = 9,000,000 / 1,000,000 = 9 次。毫秒系数 = 9 × 1000 = 9000。

3. 微秒延时 — DelayUs

实现 1 ~ 798,915 微秒的精确延时。

void DelayUs(unsigned short us)
{
    unsigned int ctrlResult = 0;

    us &= 0x00FFFFFF;                           // 仅取低 24 位（计数器宽度）

    SysTick->LOAD = us * UsCount;               // 设置重装载值
    SysTick->VAL  = 0;                             // 清零当前计数值
    SysTick->CTRL = 1;                             // 使能计数器（ENABLE=1）

    do {
        ctrlResult = SysTick->CTRL;                // 读取控制寄存器
    } while ((ctrlResult & 0x01) && !(ctrlResult & (1 << 16)));
    // 条件：计数器使能位为 1 且 COUNTFLAG 尚未置位

    SysTick->CTRL = 0;                             // 关闭计数器
    SysTick->VAL  = 0;                             // 清零计数值
}
注意： 参数 us 被截断为 24 位，最大支持约 798,915 μs（约 798 ms）。若需更长的延时请调用 DelayXms。

4. 毫秒延时 — DelayXms

实质是对微秒延时的"放大"版，使用 MsCount = UsCount × 1000 直接装载毫秒计算值，原理与 DelayUs 相同。

void DelayXms(unsigned short ms)
{
    unsigned int ctrlResult = 0;

    if (ms == 0) return;                        // 参数保护

    ms &= 0x00FFFFFF;                             // 取低 24 位

    SysTick->LOAD = ms * MsCount;                // 装载毫秒计数值
    SysTick->VAL  = 0;
    SysTick->CTRL = 1;                             // 使能计数器

    do {
        ctrlResult = SysTick->CTRL;
    } while ((ctrlResult & 0x01) && !(ctrlResult & (1 << 16)));

    SysTick->CTRL = 0;
    SysTick->VAL  = 0;
}

5. 工作原理

SysTick 是 Cortex-M3 内核内置的 24 位递减计数器，以下时序图说明一次延时的工作流程：

1. 设置 LOAD = us × UsCount（例如延时 10 μs ⇒ LOAD = 10 × 9 = 90）
2. 写 VAL = 0 清零当前计数值
3. 写 CTRL = 1 使能计数器，SysTick 从 LOAD 开始递减
4. 每经过 1 个时钟周期（~111 ns），VAL 减 1
5. 当 VAL 递减至 0 时，硬件自动将 COUNTFLAG（CTRL 的 bit 16）置 1
6. 软件轮询等待 COUNTFLAG 为 1
7. 写 CTRL = 0 关闭计数器，写 VAL = 0 清理，完成延时 
// 寄存器操作拆解
SysTick->CTRL     // 0xE000E010  控制寄存器
  // bit 0: ENABLE     — 1=使能, 0=关闭
  // bit 1: TICKINT    — 中断使能（本模块不使用）
  // bit 2: CLKSOURCE  — 0=HCLK/8, 1=HCLK
  // bit 16: COUNTFLAG — 只读, 计数到 0 时硬件置 1

SysTick->LOAD      // 0xE000E014  重装载值寄存器（24 位有效）
SysTick->VAL       // 0xE000E018  当前计数值寄存器
寄存器地址本模块操作
CTRL0xE000E010初始化清 bit 2；使能时写 1；轮询 bit 16；结束后写 0
LOAD0xE000E014写入 us × UsCount 或 ms × MsCount
VAL0xE000E018启动前写 0，结束后写 0
为什么不用中断？ 本模块采用轮询模式而非 SysTick 中断，因为延时场景下 CPU 本身就需要等待，无需中断介入。这也避免了中断抢占导致的延时误差。轮询同时检查 CTRL & 0x01 确保计数器仍在运行状态，防止因意外关闭导致死循环。

#include "bsp_delay.h"

static unsigned char  UsCount = 0;
static unsigned short MsCount = 0;

/**
  * @brief  SysTick 初始化
  * @param  无
  * @retval 无
  */
void Delay_Init(void)
{
    // 选择时钟 HCLK/8 (72MHz/8=9MHz)
    SysTick->CTRL &= ~(1 << 2);

    UsCount = 9;         // 1us 需要 9 个计数
    MsCount = UsCount * 1000;  // 1ms 需要 9000 个计数
}

/**
  * @brief  微秒级延时
  * @param  us：延时微秒数（最大约 798,915 us）
  * @retval 无
  */
void DelayUs(unsigned short us)
{
    unsigned int ctrlResult = 0;

    us &= 0x00FFFFFF;

    SysTick->LOAD = us * UsCount;
    SysTick->VAL  = 0;
    SysTick->CTRL = 1;

    do {
        ctrlResult = SysTick->CTRL;
    } while ((ctrlResult & 0x01) && !(ctrlResult & (1 << 16)));

    SysTick->CTRL = 0;
    SysTick->VAL  = 0;
}

/**
  * @brief  毫秒级延时
  * @param  ms：延时毫秒数
  * @retval 无
  */
void DelayXms(unsigned short ms)
{
    unsigned int ctrlResult = 0;

    if (ms == 0) return;

    ms &= 0x00FFFFFF;

    SysTick->LOAD = ms * MsCount;
    SysTick->VAL  = 0;
    SysTick->CTRL = 1;

    do {
        ctrlResult = SysTick->CTRL;
    } while ((ctrlResult & 0x01) && !(ctrlResult & (1 << 16)));

    SysTick->CTRL = 0;
    SysTick->VAL  = 0;
}
